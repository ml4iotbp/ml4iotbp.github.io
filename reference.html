ML4IoTBP DSL — Grammar Reference (JSON-Schema)

This page documents the YAML grammar defined by the JSON-Schema file (IoT-BP-ML-Grammar.json). It summarizes the root structure, the available sections, and each schema definition, including constraints (required fields, enums, regex keys, and oneOf/anyOf rules).

Overview

What this DSL covers: process context & process data sources, IoT data sources, feature definitions, dataset specification, ML models and training schedules, and runtime prediction integration.

Schema draft: http://json-schema.org/draft-07/schema#

Root document

type: object additionalProperties: false

Title: Data sources YAML grammar

Presence rules (anyOf)

The root document is valid if it contains at least one of the following required groups:

  • process-data-sources, process-context
  • iot-data-sources
  • features
  • dataset
  • models
  • runtime-predictions

Top-level sections

Typical file layout

Show an illustrative YAML skeleton
process-context:
  engine: Camunda
  host: http://localhost:8080
  process-id: smartlogistics

process-data-sources:
  task_event_source:
    format: application/json
    elements:
      evaluateQualityTask:
        type: User Task
        events:
          started:
            variables:
              - currentTime: timeStamp
                internal-id: evalStartingTime
          completed:
            variables:
              - currentTime: timeStamp
                internal-id: evalCompleteTime

iot-data-sources:
  container_temperature_sensor:
    paradigm: HTTPCompliant
    format: application/json
    end-point: http://sensor/api/temp
    method: GET
    sampling-time: 10s
    schema:
      - timestamp: dateTime
      - temperature: float

features:
  f_avg_temp:
    from: container_temperature_sensor
    operation: avg
    field: temperature
    window: 30m
    anchor:
      element: evaluateQualityTask
      event: completed

dataset:
  microbial_risk_ds:
    train-ratio: 0.7
    val-ratio: 0.15
    test-ratio: 0.15
    data:
      - PI: f_pallet_id
      - CTB: f_avg_temp

models:
  ml_bacteria_prediction:
    algorithm: Random Forest
    learning-type: Supervised
    dataset: microbial_risk_ds
    training:
      type: Batch
      metrics: [f1, roc_auc]
      schedule:
        type: Periodic
        every: 24h

runtime-predictions:
  predict_bacteria:
    model: ml_bacteria_prediction
    trigger:
      type: Periodic
      every: 10m
      first:
        evaluateQualityTask: completed
    inputFeatures: [f_avg_temp]
    output: {}

Definitions

Each definition corresponds to a reusable grammar fragment referenced from one or more sections.

Runtime Predictions (runtime-predictions)

type: object additionalProperties: false

Pattern properties

RegexSchema
^[A-Za-z_][A-Za-z0-9_-]+$type: object
Usage hint: A runtime prediction references a model, a trigger schedule, inputFeatures, and an output block.

Models (models)

type: object additionalProperties: false

Pattern properties

RegexSchema
^[A-Za-z_][A-Za-z0-9_-]+$type: object
Usage hint: A model references a dataset and provides algorithm, learning-type, training type, optional metrics and a schedule (Periodic vs Event-Based).

Dataset (dataset)

type: object additionalProperties: false

Pattern properties

RegexSchema
^[A-Za-z_][A-Za-z0-9_-]+$type: object
Usage hint: A dataset entry can optionally define a label (supervised) and a sampling-point anchor. data is an array with at least one item.

Variable (variable)

type: object additionalProperties: false

A variable is represented as an object with one mandatory property: { <varName>: <varTypec> } and an optional internal_id.
